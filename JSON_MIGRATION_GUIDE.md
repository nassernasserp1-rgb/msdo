# 🔄 دليل تحويل ملف JSON القديم

## المشكلة التي تواجهها:
السكرابر المحسن الجديد يستخدم قاعدة بيانات SQLite بدلاً من ملف JSON، لذلك لا يقرأ ملف JSON القديم تلقائياً.

## 💡 الحلول المتاحة:

### الحل الأول: تحويل JSON إلى SQLite (الأسرع) ⚡
```bash
# تشغيل أداة التحويل التلقائي
python migrate_json_to_db.py

# أو تحديد ملف JSON يدوياً
python migrate_json_to_db.py path/to/your/file.json
```

**المميزات:**
- ✅ تحويل كامل لجميع البيانات
- ✅ يحافظ على تاريخ الأسعار
- ✅ أداء فائق مع SQLite
- ✅ توافق كامل مع السكرابر المحسن

### الحل الثاني: استخدام السكرابر المتوافق مع JSON 🔗
```bash
# تشغيل السكرابر الذي يقرأ JSON مباشرة
python json_compatible_scraper.py

# أو استخدام الواجهة الرسومية المطورة
python integrated_app_with_json.py
```

**المميزات:**
- ✅ يقرأ ملف JSON القديم مباشرة
- ✅ يدمج البيانات القديمة مع الجديدة
- ✅ لا يحتاج تحويل مسبق
- ✅ واجهة رسومية محسنة

## 🚀 استخدام السكرابر المتوافق مع JSON:

### في الواجهة الرسومية:
1. شغل: `python integrated_app_with_json.py`
2. اضغط "🔍 اختيار ملف JSON"
3. اختر ملف JSON الخاص بك
4. ابدأ السكرابة كالمعتاد

### في الكود مباشرة:
```python
import asyncio
from json_compatible_scraper import JSONCompatibleScraper

async def main():
    # تحديد ملف JSON
    async with JSONCompatibleScraper(json_file="your_products.json") as scraper:
        await scraper.scrape_section_optimized(
            section="Electronics",
            base_url="https://amazon.eg/...",
            start_page=1,
            end_page=10,
            discount_threshold=30.0
        )
        
        # حفظ البيانات المدمجة
        scraper.save_combined_data("updated_products.json")

asyncio.run(main())
```
